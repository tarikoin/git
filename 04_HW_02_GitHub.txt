#1. На локальном репозитории сделать ветки для:
#- Postman
git branch Postman

#- Jmeter
git branch Jmeter

#- CheckLists
git branch Checklists 

#- Bag Reports
git branch BugReposts
 
#- SQL
git branch SQL 

#- Charles
git branch Charles 

#- Mobile testing
git branch MobileTesting

#2. Запушить все ветки на внешний репозиторий
git push origin --all

#3. В ветке Bag Reports сделать текстовый документ со структурой баг репорта
git checkout BugReports
cat > bug_report.txt
Bug Title: Поле количества единиц товара в корзине допускает ввод нечисловых значений
Bug ID: WB-0001
Environment: Google Chrome Версия 100.0.4896.127 (Официальная сборка), (64 бит)
Priority: Medium
Reported By:    Anna
Reported On:    2022/06/18
Type:   Bug
Status: New
Description:    Поле количества единиц товара в корзине допускает ввод нечисловых значений и оформление заказа с этими значениями
Steps To Reproduce:     
    1. Добавить товар в корзину (например: https://www.wildberries.ru/catalog/23018620/detail.aspx?targetUrl=XS); 
    2. Перейти в корзину; 
    3. В поле количества единиц товара ввести любое нечисловое значение (например: ФФ)
Expected Result: Нечисловое значение не вводится в поле количества товара либо появляется сообщение об ошибке
Actual Result:  Нечисловое значение вводится и сохраняется в поле количества товара
Ctrl+C

#4. Запушить структуру багрепорта на внешний репозиторий
git add bug_report.txt
git commit -m "Bug report structure"
git push

#5. Вмержить ветку Bag Reports в Main
git checkout main
git merge BugReports -m "new branch merge"

#6. Запушить main на внешний репозиторий.
git push origin main

#7. В ветке CheckLists набросать структуру чек листа.
git checkout CheckLists
cat > check_list_structure.txt
1. Uploading files to the library
    1.1. txt
    1.2. jpg
    1.3. pdf
2. Downloadig files from the library
    2.1. txt
    2.2. jpg
    2.3. pdf
Ctrl+C

8. Запушить структуру на внешний репозиторий
git add check_list_structure.txt
git commit -m "Check List structure"
git push origin CheckLists

#9. На внешнем репозитории сделать Pull Request ветки CheckLists в main
#Pull requests -> CheckLists -> Compare & pull request -> Create pull request -> Confirm merge

#10. Синхронизировать Внешнюю и Локальную ветки Main
git checkout main
git fetch
git pull
