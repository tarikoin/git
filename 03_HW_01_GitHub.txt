#JSON
#4. Создать внешний репозиторий c названием JSON.
Repositories -> New -> JSON

#5. Клонировать репозиторий JSON на локальный компьютер.
git clone git@github.com:tarikoin/JSON.git

#6. Внутри локального JSON создать файл “new.json”.
touch new.json

#7. Добавить файл под гит.
git add new.json

#8. Закоммитить файл.
git commit -m "the first json created"

#9. Отправить файл на внешний GitHub репозиторий.
git push

#10. Отредактировать содержание файла “new.json” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате JSON.
$ cat >> new.json
{
     name: "Anna",
     age: 35,
     pets: 1,
     salary: 1000
}
Ctrl+C

#11. Отправить изменения на внешний репозиторий.
git add new.json
git commit -m "file modified"
git push

#12. Создать файл preferences.json
touch preferences.json

#13. В файл preferences.json добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате JSON.

     season: "spring",

    <TITLE>Поле количества единиц товара в корзине допускает ввод нечисловых значений</TITLE>
    <ID>WB-0001</ID>
    <ENVIRONMENT>Google Chrome Версия 100.0.4896.127 (Официальная сборка), (64 бит)</ENVIRONMENT>
    <PRIORITY>Medium</PRIORITY>
    <REPORTED_BY>Anna</REPORTED_BY>
    <REPORTED_ON>2022/06/18</REPORTED_ON>
    <TYPE>Bug</TYPE>
    <STATUS>New</STATUS>
    <DESCRIPTION>Поле количества единиц товара в корзине допускает ввод нечисловых значений и оформление заказа с этими значениями"</DESCRIPTION>
    <STEPS>
        <STEP>1. Добавить товар в корзину (например: https://www.wildberries.ru/catalog/23018620/detail.aspx?targetUrl=XS)</STEP>
        <STEP>2. Перейти в корзину</STEP>
        <STEP>3. В поле количества единиц товара ввести любое нечисловое значение (например: ФФ)"</STEP>
    </STEPS>
    <EXPECTED_RESULT>Нечисловое значение не вводится в поле количества товара либо появляется сообщение об ошибке</EXPECTED_RESULT>
    <ACTUAL_RESULT>Нечисловое значение вводится и сохраняется в поле количества товара</ACTUAL_RESULT>
</REPORT>
